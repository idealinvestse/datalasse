# OpenClaw Workspace

> The home of an autonomous AI agent. Not a product. Not a library.
> A place to live, work, and remember.

This repository is the working directory of an [OpenClaw](https://github.com/openclaw/openclaw) agent
running on `vps-agent-1`. It contains the agent's skills, scripts, examples, and
local notes — but **not** its long-term memory, secrets, or runtime state.

For the full operating contract, see **[AGENTS.md](./AGENTS.md)**.

## What's deliberately excluded

This is a personal agent workspace. A few classes of files live in the
working tree but are **not** in git:

- **`MEMORY.md` + `memory/`** — the agent's long-term memory and daily
  session logs. Loaded only in the main (direct) session per `AGENTS.md`.
- **Auto-generated runtime snapshots** — `STATUS.md`, `STATUS.json`,
  VPS-research logs, gateway state. Regenerated by cron, never the
  source of truth.
- **Secrets and credentials** — `api_config.py`, `groq_wrapper.py`,
  any `.env*` file (only `.env.example` is committed).
- **Internal strategy** — `decisions.md`, `STATUS_RESEARCH.md`,
  `config-audit-report.md`.
- **Symlinked project repos** — `projects/` points at repos with
  their own `.git`. Don't nest.
- **One-shot research output** — `research/` (large timestamped
  catalogs). Source-of-truth lives in `memory/research/`.
- **Build artifacts** — `__pycache__/`, `.pytest_cache/`,
  `node_modules/`, `*.log`, `*.pid`, `*.bak`, editor swap files.

See `.gitignore` for the full list with comments.

## Running the workspace

This is a **runtime artifact**, not a deployable. The intended
consumers are:

- The agent itself, which reads `AGENTS.md`, `SOUL.md`, `TOOLS.md`, etc.
  at session start.
- The human owner, who can `git diff` to review what the agent changed.
- Future agents on a fresh VPS, who can clone this repo as a
  bootstrap template (after replacing `MEMORY.md`, `USER.md`, secrets,
  and `projects/` symlinks with their own).
